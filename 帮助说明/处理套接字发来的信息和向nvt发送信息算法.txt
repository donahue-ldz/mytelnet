打印从套接字接收到的单个字符
打印从套接字接收到的单个字符由main_fsm.c中的ttputc函数实现。主有限状态机调用此过程打印从套接字接受到的单个字符。其算法如下：
1. 如果字符未经NVT编码（二进制传输），则直接打印；
2. 如果当前在同步状态(synching非零)，则不显示字符；
3. 如果回车(CR)和换行(LF)同时出现则作为Unix系统的换行处理；
4. 如果出现单个回车或换行或其他特殊字符，则调用tcout完成该字符相关联的动作；
5. 符合打印的条件(c >= ' ' && c < TCIAC)，则直接打印。






向服务器发送数据时NVT的实现
向服务器发送字符由key_fsm.c中的soputc函数实现。套接字输出有限状态机调用过程soputc将键盘输入字符转换成网络虚拟终端编码，并通过TCP套接字将编码发送给服务器。其算法如下：
1. 如果定义为二进制传输，遇到IAC字符则进行字符填充，其它所有字符直接发送；
2. 使用NVT编码时，把要发送的字符c与0x7f进行位与后赋给字符c，保证发送字符为7字节的ASCII编码；
3. 遇到特殊字符时，进行相应的转换，并发送转换后的字符序列；
4. 非特殊字符则调用putc发送出去。
说明：
NVT协议规定，如果服务器不请求客户使用二进制传输，服务器将丢弃用户键入的大多数控制字符。
